Final Project Overview

Introduction

We successfully implemented an ETL (Extract, Transform, Load) pipeline that achieves data interoperability between systems such as OpenEMR, and a Primary Care EHR. This project solves the challenge of seamlessly exchanging patient information, their conditions, observations, and procedures and adhering to FHIR (Fast Healthcare Interoperability Resources) standards at the same time. We have extracted data from OpenEMR, transformed it using SNOMED CT terminologies and loaded it into the target system - Primary Care EHR. This project uses Python and REST APIs to automate the data exchange between healthcare systems.

Purpose

The purpose of this ETL pipeline is to integrate and transform the data between multiple systems - OpenEMR FHIR Server, Hermes Terminology Server, and the Primary Care EHR FHIR Server. This pipeline facilitates data exchange and management of patient information.
